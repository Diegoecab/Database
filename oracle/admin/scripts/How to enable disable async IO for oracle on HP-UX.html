How to enable/disable async I/O for oracle on HP-UX “Ioctl ASYNC_CONFIG error, errno = 1 “

May 18th, 2010 admin Leave a comment Go to comments

Reading time: 2 – 2 minutes

IF you get the error  “Ioctl ASYNC_CONFIG error, errno = 1 ” in tracefile. :
The “Ioctl ASYNC_CONFIG error, errno = 1″ Error means that the asynchronus IO is not configured correctly on the HP-UX Platform.

To Disable async IO at oracle level ( To avoid this error ) :
++ Since we have a problem with async I/O, we have used synchronous I/O to perform the expdp by doing following changes in init.ora.
disk_asynch_io=FALSE
filesystemio_options=none

To Enable async IO at HP-UX and Oracle level ( To avoid this error ) :
1- $ ls -l /dev/async
The /dev/async device should have the following permissions:
# chown oracle:dba /dev/async
# chmod 660 /dev/async

2- AIO , will not work for Oracle, unless the dba group has the MLOCK privilege.
So we need to check the following :

a- To check if the dba group has the MLOCK privilege, execute:
# /usr/bin/getprivgrp dba

b- If the dba group does not have the MLOCK privilege then set it for group dba
% /usr/sbin/setprivgrp dba MLOCK

c- Note, the next reboot will clear this privilege. To automate this at startup,

create /etc/privgroup with the entry
dba MLOCK

This can be tested with
# /usr/sbin/setprivgrp -f /etc/privgroup

3- For more information about HP-UX AIO configurations , Please check the following note :
Note 139272.1 Ext/Pub HP-UX Asynchronous i/o

4- After verifying and correcting all the AIO configurations , Then please set in your parameter file :
disk_async_io = true
filesystemio_options=setall

5- shutdown the instance

6- startup the instance

7- Done!
